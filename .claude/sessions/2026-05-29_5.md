## 의사결정 기록

### 채택한 결정

1. **자동완성 전 시군구 필터 선행**
   - 이유: "롯데캐슬", "자이", "래미안" 등 전국에 수십 개인 동명 단지는 자동완성 결과가 구분 불가. 시군구로 먼저 범위를 좁히면 드롭다운이 실용적 크기로 유지됨.
   - 검토된 대안 A — 자동완성 결과에 `(시군구)` 표기만 추가: 드롭다운 항목이 너무 많고 사용자가 목표를 스크롤로 찾아야 함 → 기각.
   - 검토된 대안 B — 현재 검색 버튼 방식 유지 + 결과 목록에서 선택: 이미 있던 방식이지만 동명 단지 혼란을 해소하지 못함 → 기각.

2. **기존 검색 버튼·결과 목록 제거 (자동완성으로 통합)**
   - 이유: 시군구 선택 후 자동완성 드롭다운이 결과 목록 역할을 대체. 버튼·목록이 남으면 UX 흐름이 중복됨.

3. **외부 클릭 감지에 `onMouseDown` + ref 방식 사용**
   - 이유: input `onBlur` → `onClick` 순서 문제로 드롭다운이 닫히기 전 클릭 이벤트가 소실될 수 있음. `onMouseDown`은 blur보다 먼저 발생해 항목 클릭이 보장됨.

4. **`/api/search` — sgg 파라미터 선택적 적용**
   - 이유: 기존 전국 검색 동작 유지 + 시군구 한정 검색 추가. 기존 `apt+sgg` 이력 쿼리 분기보다 뒤에 위치하므로 충돌 없음.

### 검토했지만 채택하지 않은 대안

| 대안 | 기각 이유 |
|---|---|
| 자동완성만 구현 (시군구 필터 없이) | 동명 단지 수십 개 드롭다운 — 비효율적이라는 사용자 직관에 동의. 시군구 선행이 UX상 올바름 |
| 자동완성 미구현, 시군구 필터만 추가 + 엔터 검색 | 사용자가 자동완성까지 요청했으므로 기각 |
