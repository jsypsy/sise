## 의사결정 기록

- **이미지 렌더에 `html-to-image`(클라 1개 의존성) 채택, `next/og`(내장) 미채택.** 이유: next/og는 서버 렌더라 임의 한글 아파트명 전체 글리프 폰트(노토 1~2MB) 번들 필요 + 카드 디자인 중복 작성. html-to-image는 브라우저 폰트 그대로(한글 공짜) + **서버/DB 호출 0**(클라 캡처) + 기존 카드 재사용 → 오히려 무료 티어에 안전(CLAUDE.md #5 부합). MIT 라이선스 확인(AGPL 회피 규칙 준수).
- **공유 이미지 마스트헤드 유지 + 화면에선 분리.** 이유: 떠도는 이미지에 브랜드/출처가 박혀야(성장 엔진) 하지만, 페이지엔 사이트 헤더와 중복되어 "타이틀 두 번" 버그처럼 보임 → off-screen 별도 DOM으로 해결.
- **R2 동결 해소책: 처음엔 매일 순환 스케줄(`1a2722c`)로 했으나 철회.** 이유: API quota(1만 콜)만 보고 Actions '분' 축을 못 봄. 회당 70분×매일 = 월 2,000분 초과. → 스케줄 제거 후 **7/1 이후 점진 수집(최근 N개월만 받아 R2 병합)** 으로 재설계하기로 보류.
