## 의사결정 기록

### 채택한 결정

1. **"오늘의 시그널" 완전 폐기, 7일 TOP으로 대체**
   - 이유: 국토부 신고 지연(30일 이내 신고) 구조상 "오늘 deal_date" = 3건 수준으로 무의미.
     최근 7일 범위로 확장하면 건수 확보 + "오늘 뭐 터졌나" 니즈 충족.
   - `today-client.tsx` (194줄)·`api/signals/route.ts` 삭제로 코드도 단순해짐.

2. **`/top` 전면 client component 전환 (ISR 포기)**
   - 이유: 지역 필터가 interactive해야 하므로 server ISR 불가. 대신 supabase anon 클라이언트를
     브라우저에서 직접 호출 (NEXT_PUBLIC_* 키는 이미 공개). 새 Route Handler 추가 없이 해결.
   - 무료 티어: 각 필터 변경 시 signals_mv 쿼리 1회 추가. 건당 수 KB로 egress 영향 미미.

3. **메인 검색은 네비게이션만, 자동완성 없음**
   - 이유: 자동완성은 시군구 선택이 선행돼야 하는 구조. 메인에서 sgg 없이 자동완성하면
     전국 ILIKE 쿼리가 매 키 입력마다 발생 → 무료 egress 낭비. 단순 form submit으로 충분.

4. **리모트에서 pull --rebase로 흡수**
   - 5/31 KST에 리모트에서 직접 작업된 커밋 4개 발견:
     - `ee0759e` fetch-peaks R2 적재 + 8샤드 전환 (Supabase Storage → Cloudflare R2)
     - `2c7a96c` lib/refine.ts 분리 (ingest main() 자동 실행 버그 수정)
     - `4495af7` 이메일 발송 추가 → `6730b00` 바로 revert
   - 충돌 없이 rebase 성공.
