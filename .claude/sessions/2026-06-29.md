## 의사결정 기록

- **동명단지 discriminator = `umd_nm`** (apt_seq 아님). 이유: 진단상 `dongs==seqs`로 umd가 완벽 분리, 이미 앱 전반에 쓰여 사람이 읽기 좋음. apt_seq는 결측(null) 시 묶임 위험. *대안 기각: apt_seq(결측 리스크).*
- **동명단지는 Layer 1(시그널)만, Layer 2(앱 URL/라우트)는 보류.** 이유: 영향이 이름 흔한 지방 저거래 단지에 한정, Layer 2는 라우트·canonical·watchlist·SEO 대규모 리팩터(AdSense 심사 중 위험). *대안 기각: 전체 리팩터(과대).*
- **hp 재구축을 MOLIT 아닌 R2에서.** 이유: MOLIT 전국 풀수집은 일일 quota+6h 타임아웃으로 실패 확정. R2엔 전기간 거래가 이미 있어 quota 0·22분. *트레이드오프: R2는 per-deal umd가 없어 동명단지 baseline이 보수적 → 밤 MOLIT 샤드가 점진 보정(수용).* *대안 기각: MOLIT 샤드 8일 풀(8일간 신고가 과다).*
- **self-inclusion 가드(PEAK_CUTOFF):** hp에 라이브 윈도우(최근 3개월) 제외. 이유: signals_mv가 그 구간을 transactions에서 직접 계산하므로 hp에 넣으면 현재월 거래가 자기 전고점이 돼 진짜 신고가 억제. 수동/스케줄 hp 결과 동일 → 밤 cron이 재구축을 안 되돌림.
- **GA는 `next/script`로(=@next/third-parties 미도입).** 이유: CLAUDE.md 규칙2(현재 스택 우선·새 의존성 최소). *대안 기각: @next/third-parties(불필요한 dep + lock 갱신).*
- **GA 측정 ID는 코드 기본값으로 하드코딩**(`SITE_URL` 패턴). 이유: 측정 ID는 클라이언트 HTML 노출 공개값(비밀 아님) → 레포에 둬도 안전, env 설정 없이 즉시 작동. env override 유지. *CLAUDE.md 5번 관련: service_role과 달리 NEXT_PUBLIC 노출 무방.*
- **CLAUDE.md 5번 불변 아키텍처 준수:** 수집은 GitHub Actions(Vercel cron 미추가), 시그널은 Postgres 뷰, ISR 유지, service_role은 Actions/psql 전용(GA_ID는 공개값이라 별개).
